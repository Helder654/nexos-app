# NexOS

O NexOS é um aplicativo Android desenvolvido como projeto final da disciplina de Programação para Dispositivos Móveis do curso de Análise e Desenvolvimento de Sistemas.

A proposta do projeto é auxiliar assistências técnicas de videogames no gerenciamento de ordens de serviço, permitindo acompanhar todo o processo, desde o cadastro do cliente até a entrega do equipamento.

Durante o desenvolvimento utilizei Kotlin, Jetpack Compose e Firebase, buscando aplicar na prática os conceitos aprendidos ao longo da graduação.

Como todo projeto acadêmico, ele foi desenvolvido dentro do tempo disponível para a disciplina. Algumas funcionalidades que imaginei inicialmente ficaram de fora, mas pretendo continuar evoluindo o NexOS como forma de praticar e aprender ainda mais.

---

## Funcionalidades

Atualmente o aplicativo conta com:

- Autenticação de usuários
- Cadastro de clientes
- Cadastro de consoles
- Gerenciamento de ordens de serviço
- Controle do status das OS
- Controle de garantia
- Suporte para múltiplos técnicos
- Relatórios

---

## Tecnologias utilizadas

- Kotlin
- Jetpack Compose
- Firebase Authentication
- Cloud Firestore
- Material Design 3
- Android Studio

---

## Estrutura do projeto

O projeto foi organizado buscando separar as responsabilidades de cada parte da aplicação, facilitando sua manutenção e evolução.

A estrutura principal está dividida entre:

- Telas (Screens)
- Componentes reutilizáveis
- Modelos
- Serviços
- Navegação
- Utilitários

---

## Como executar

1. Clone o repositório.

```bash
git clone https://github.com/Helder654/nexos-app.git
```

2. Abra o projeto no Android Studio.

3. Configure um projeto no Firebase.

4. Adicione o arquivo `google-services.json` na pasta `app`.

5. Execute o aplicativo em um dispositivo ou emulador Android.

## O que aprendi

Esse foi o maior projeto Android que desenvolvi durante a graduação.

Além de aprofundar meus conhecimentos em Kotlin e Jetpack Compose, aprendi bastante sobre organização de projetos, integração com Firebase, gerenciamento de estado e resolução de problemas encontrados durante o desenvolvimento.

Também foi um projeto importante para entender como funciona o desenvolvimento de uma aplicação um pouco maior do que os exercícios feitos em sala de aula.
